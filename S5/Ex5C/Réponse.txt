Nom du Pattern : Composite
Participants théoriques 		Noms utilisés dans le code
	IComponent				Partie
	Composite				Groupe
	Leaf					Valeur

Nom théorique des méthodes 		Nom utilisé dans le code
	operation()			demande(), getNiveau()

Nom du Pattern : Iterator
Participants théoriques 		Noms utilisés dans le code
	Aggregate				Groupe
	Iterator				Iterator

Nom théorique des méthodes 		Nom utilisé dans le code
	iterator()				getParties()

Nom du Pattern : Visitor
Participants théoriques 		Noms utilisés dans le code
	Visitor					Traitement
	ConcreteVisitor				Listeur, Totaliseur, ListeurNom
	Element					Partie
	ConcreteElement				Valeur, Groupe
	Client					Main

Nom théorique des méthodes 		Nom utilisé dans le code
	visit(ConcreteElement)			traiteValeur(Valeur), traiteGroupe(Groupe)
	accept(Visitor)				demande(Traitement)

	ConcreteCreator				Fabriquant

Nom théorique des méthodes 		Nom utilisé dans le code
